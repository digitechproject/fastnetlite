Configuration du Walled Garden IP Mikrotik

Pour permettre à vos clients d'accéder à FastNetLite depuis la page de connexion du hotspot Mikrotik, vous devez ajouter les règles suivantes dans votre routeur Mikrotik. Ces règles permettent d'autoriser l'accès à FastNetLite et aux services associés (Firebase, FedaPay, etc.) sans nécessiter d'authentification.

Les commandes ci-dessous sont prêtes à être copiées et collées dans votre routeur Mikrotik. 
                # Règles pour le domaine principal hébergeant FastNetLite
                /ip hotspot walled-garden ip 
                add action=accept  comment="FASTNETLITE" dst-host=fastnetlite.vercel.app dst-port=443 protocol=tcp
                add action=accept  comment="FASTNETLITE" dst-host=fastnet.itikets.com dst-port=443 protocol=tcp 
                                        
                # Règles pour FedaPay (système de paiement)
                add action=accept comment="FedaPay Sandbox API" dst-host=sandbox-api.fedapay.com
                add action=accept comment="FedaPay Sandbox Process" dst-host=sandbox-process.fedapay.com
                add action=accept comment="FedaPay Sandbox Checkout" dst-host=sandbox-checkout.fedapay.com
                add action=accept comment="FedaPay Live API" dst-host=api.fedapay.com
                add action=accept comment="FedaPay Live Process" dst-host=process.fedapay.com
                add action=accept comment="FedaPay Live Checkout" dst-host=checkout.fedapay.com

                # Règles pour Firebase (base de données et authentification)
                add action=accept comment="FIREBASEIO" dst-host=*.firebaseio.com
                add action=accept comment="FIRESTORE API" dst-host=firestore.googleapis.com
                add action=accept comment="FIREBASEAPP" dst-host=*.firebaseapp.com
                add action=accept comment="FIREBASE API" dst-host=firebase.googleapis.com
                add action=accept dst-host=identitytoolkit.googleapis.com

                # Règles pour les CDN utilisés par la page
                add action=accept dst-host=cdn.jsdelivr.net
                add action=accept dst-host=cdnjs.cloudflare.com
Après avoir ajouté ces règles, redémarrez le service hotspot de votre routeur Mikrotik pour appliquer les changements.
